# Needed Implementations & Workflow Enhancements

Based on the analysis of the current "Surplus Food Donation" application, the following gaps have been identified. Implementing these will transition the project from a basic prototype to a secure, functional platform.

## 1. Security Enhancements (Critical)
| Gap | Severity | Recommendation |
|---|---|---|
| **Plaintext Passwords** | High | Use `BCryptPasswordEncoder` to hash passwords before saving them in the `users` table. |
| **Route Protection** | High | Implement **Spring Security** or a **WebInterceptor** to prevent unauthorized users from accessing `/admin-dashboard`, `/donor-dashboard`, etc., via direct URL. |
| **Session Tracking** | Medium | Add a `/logout` method in `AuthController` to invalidate the session and clear security context. |

## 2. Workflow Logic Gaps
### A. Donation Lifecycle Management
- **Status Field**: Add a `status` field (Enum: `PENDING`, `ACCEPTED`, `PICKED_UP`, `COMPLETED`, `EXPIRED`) to the `Donation` model.
- **Acceptance Logic**: Extend the `Donation` model to include a `ManyToOne` relationship with `User` (NGO) to track who accepted the food.
- **Automated Expiry**: Implement a `@Scheduled` task in Spring Boot to automatically mark donations as `EXPIRED` if the current time exceeds the `expiryTime`.

### B. Communication & Notifications
- **Email/SMS Integration**: Notify NGOs when a new donation is posted within a specific radius.
- **Donor Updates**: Notify Donors via email when their food is accepted or successfully picked up.

## 3. Feature Suggestions (Modernization)
- **Image Upload**: Integrate **MultipartFile** support in the `addSurplusfood` form to allow donors to upload photos of the food for quality verification.
- **Dynamic Maps**: Use the Google Maps JavaScript API to fetch long/lat from the `pickupAddress` and plot real-time markers on the `/view-map` page.
- **Rating & Review System**: Allow NGOs to rate Donors (on food quality) and Donors to rate NGOs (on pickup punctuality) to build community trust.
- **Quantity Tracking**: If an NGO only needs half the quantity, allow partial pickups instead of closing the entire donation.

## 4. Technical Debt
- **Error Handling**: Replace generic return types with a global `@ControllerAdvice` to handle database exceptions and 404/500 errors gracefully.
- **Data Validation**: Add `@NotBlank`, `@Size`, and `@Email` annotations to the `User` and `Donation` models to prevent invalid data entry from the frontend.
